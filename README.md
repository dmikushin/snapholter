# SnapECG Holter

Android-приложение для длительного мониторинга ЭКГ с платой SnapECG B10 по Bluetooth LE.

## Возможности

- Запись ЭКГ 200 Гц в SQLite во время измерения
- Детекция QRS (алгоритм Pan-Tompkins) и отображение BPM
- Уведомление ongoing с длительностью записи, BPM, батареей
- Опциональное подключение к PC Connector для AI-ассистенции
- Экспорт записей в CSV

## LEAD OFF — потеря контакта электрода

Во время записи периодически появляется предупреждение **⚠ LEAD OFF** на экране и в уведомлении. Это **не баг**, а особенность железа SnapECG B10.

**Причина**: чип B10 аппаратно определяет электрический контакт электрода с кожей. Когда контакт пропадает (электрод отошёл, гель подсох, провод натянулся), чип выставляет флаг lead-off в пакете данных (бит 5 high-байта).

**Что происходит в этот момент**:
- Сигнал ЭКГ уходит в насыщение — данные **недостоверны**
- BPM **не обновляется** до восстановления контакта и накопления новых валидных ударов
- Запись продолжается — сэмплы пишутся как есть (будут отфильтрованы при постобработке)

**Это нормально для длительных записей**. За 12 часов неизбежны эпизоды, когда электрод смещается при движении. Если lead-off горит постоянно — проверьте:
1. Достаточно ли геля на электроде
2. Плотно ли прилегает электрод (не болтается)
3. Не перетёрся ли провод в месте пайки к плате

## Сборка

Сборка идёт внутри Docker — нужен установленный и запущенный `docker`.

Первый раз: собрать образ с Android SDK (~ 5 минут):
```bash
cd snapecg-holter
docker build -t snapecg-holter .
```

Сборка APK и прогон тестов:
```bash
./build_docker.sh                # полная сборка: тесты + APK
./build_docker.sh assembleDebug  # только APK
```

Артефакты складываются рядом со скриптом, в `snapecg-holter/.docker-build/`:
- `outputs/apk/debug/app-debug.apk` — установочный APK
- `reports/tests/testDebugUnitTest/index.html` — HTML-отчёт по unit-тестам
- `build.log` — полный лог Gradle
